# 端到端延迟测试工具 (End-to-End Latency Test)

这是一个用于测试机器视觉系统（如 RK3588 开发板）端到端反应延迟的自动化 Python 脚本。

它通过在屏幕上显示特定的测试图片，并监听鼠标移动事件来计算从“显示图片”到“接收到操作”之间的时间差。

## 功能特点

*   **全自动化流程**：无需人工干预，启动后自动黑屏等待并开始测试。
*   **高精度计时**：使用 `time.time()` 记录显示帧和鼠标事件的时间戳。
*   **防误触机制**：在测试开始前一刻自动校准鼠标位置，并设置移动阈值，防止微小抖动导致误判。
*   **全屏显示**：自动全屏显示测试图片，确保视觉识别不受干扰。

## 环境要求

*   Python 3.x
*   Windows / Linux / macOS

## 安装依赖

请确保安装了以下 Python 库：

```bash
pip install opencv-python pynput numpy
```

## 使用方法

1.  **准备测试图片**：
    在脚本同级目录下放置一张名为 `test.png` 的图片。这张图片应该是你的视觉算法（如 YOLO）能够识别的目标图片。

2.  **连接设备**：
    *   确保被测设备（如 RK3588）的摄像头正对电脑屏幕。
    *   确保被测设备的模拟鼠标（HID）已连接到运行此脚本的电脑。

3.  **运行脚本**：

    ```bash
    python Timing.py
    ```

4.  **测试流程**：
    *   脚本启动后，屏幕会立即变黑。
    *   **等待 2 秒**（给设备和摄像头适应时间）。
    *   屏幕突然显示 `test.png`。
    *   脚本开始计时。
    *   一旦检测到鼠标移动（由被测设备触发），脚本停止计时并输出延迟时间。

## 配置说明

你可以在 `Timing.py` 文件顶部修改以下配置：

```python
# --- 配置 ---
IMAGE_PATH = 'test.png'     # 测试图片路径
TIMEOUT = 5.0               # 最大等待时间 (秒)
MOVEMENT_THRESHOLD = 10     # 判定为有效移动的最小距离 (像素)
```

## 常见问题

*   **一直显示超时？**
    *   检查摄像头是否能清晰看到屏幕。
    *   检查被测设备的算法是否成功识别了 `test.png`。
    *   检查模拟鼠标是否正确连接并能控制电脑光标。
*   **反应时间为 0ms 或极短？**
    *   可能是鼠标在测试开始瞬间发生了抖动。尝试增大 `MOVEMENT_THRESHOLD` 值。
